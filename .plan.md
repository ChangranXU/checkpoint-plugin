- Pass-14 MISS-5 fix at `resume.py:1208-1214` (`_codex_session_meta()`) correctly uses `is_subagent` discriminator to suppress `forked_from_id` for resume operations
- BUT resume codepath ALSO calls `_codex_head_meta()` at `resume.py:1146-1147` which unconditionally adds `forked_from_id` when `original_session_id` parameter is present
- The two functions are called from different code paths:
  - `_codex_head_meta()` → called from `_rewrite_codex_trajectory()` line ~1300 for HEAD meta
  - `_codex_session_meta()` → called from `_rewrite_codex_trajectory()` line ~1350 for body metas
- The `is_subagent` discriminator logic only exists in `_codex_session_meta()`, not `_codex_head_meta()`

**Location**: `src/checkpoint_plugin/resume.py:1146-1147`
```python
def _codex_head_meta(
    new_session_id: str,
    cwd: Path,
    source_meta: dict[str, object] | None,
    original_session_id: str | None,  # ← passed for both fork AND resume
    now: str,
) -> dict[str, object]:
    payload: dict[str, object] = {"id": new_session_id}
    if original_session_id:
        payload["forked_from_id"] = original_session_id  # ← WRONG for resume
    # ...
```

**Expected behavior**:
- Native codex startup sessions: NO `forked_from_id` in HEAD meta
- Native codex resume sessions: NO `forked_from_id` in HEAD meta (resume is like startup)
- Native codex fork sessions: YES `forked_from_id` in HEAD meta
- Native codex subagent sessions: YES `forked_from_id` in HEAD meta (inherit parent's fork ancestry)

**Fix**: Add discriminator to `_codex_head_meta()` to distinguish resume from fork, similar to `_codex_session_meta()` fix. Options:
1. Add `is_resume` boolean parameter
2. Check `source_meta.get("source")` to detect resume vs fork vs subagent
3. Pass `source_meta` and use same `is_subagent` logic as `_codex_session_meta()`
