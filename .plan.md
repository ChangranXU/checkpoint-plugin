### FORK-TRUNCATION: forked_at_offset exceeds parent file size (HIGH - KNOWN LIMITATION)

**Severity**: HIGH (data loss, fork lineage broken)

**Status**: KNOWN LIMITATION - Race condition in fork capture timing

**Affected sessions**: 4 codex fork sessions
- 019e82d0-d2c1-74e0-90ee-6730bfe583c3: offset=138581, parent=94247 bytes (short 44334 bytes)
- 019e82d2-a56d-7ee0-907e-d0e307887182: offset=183526, parent=147059 bytes (short 36467 bytes)
- 019e82dc-ad8b-79e0-b179-1c0fee08632e: offset=295415, parent=242563 bytes (short 52852 bytes)
- 019e82d2-0069-7e23-9993-4c6b9c49e54c: (not measured but likely similar)

**Root cause**: 
`_fork_anchor()` in coordinator.py:316-329 captures the parent file size at the moment the forked session's `SessionStart` hook fires. However, the parent session may still be writing/flushing data at that moment, causing the captured offset to exceed the parent's final file size.

**Why this happens**:
1. User forks a session (e.g., "fork chat" in codex)
2. Codex creates a new session and fires `SessionStart` hook
3. Plugin's `_fork_anchor()` reads parent file size → captures offset X
4. Parent session continues writing and eventually finishes at offset Y
5. If X > Y, we have a truncation issue (parent file shrinks or was never that large)

**Impact**:
- Fork sessions reference non-existent byte ranges in parent
- Resume from fork will fail or produce incorrect trajectory
- Fork lineage is broken (cannot reconstruct full conversation history)

**Potential fixes** (not implemented):
1. Add synchronization between parent and child sessions (complex, requires provider cooperation)
2. Wait for parent to finish before capturing fork anchor (but how to detect "finished"?)
3. Validate fork anchor and reject/warn if offset > parent file size
4. Lazily recompute fork anchor when parent is known to be complete

**Recommendation**: Document as known limitation. The existing recovery mechanisms (`recover_trailing_tail`, `_reanchor_last_turn_to_eof`) handle similar timing issues for turn boundaries but don't apply to fork anchors.
